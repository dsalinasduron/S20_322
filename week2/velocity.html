Planning Iterations with Velocity
  1. The first step in planning is gathering requirements.
  2. Our main tool for defining requirements are user stories.
  3. Examples of user stories.
  4. We use story points to aid in planning. Points indicate effort, not priority.
  5. Once we have decided which stories will form a deliverable, we can plan the iterations using velocity.
  6. We can plan around dates or features to decide the number of iterations and the velocity per iteration.
