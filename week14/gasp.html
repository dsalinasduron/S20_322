Generally Accepted Scrum Practices

Generally Accepted Scrum Practices are practices that are not formally part of the Scrum framework, but serve as implementation guidelines for it. For example, the backlog is part of the Scrum framework. User stories are not, but the backlog is often implemented as a collection of user stories. Generally accepted scrum practices include

  1. user stories,
  2. task boards,
  3. planning poker,
  4. burndown charts, and
  5. story maps.
